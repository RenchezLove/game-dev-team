# TEST_REPORT — отчёты тестировщика «Сборщик game-dev team» → game-lead

> Канал: **«Сборщик» (QA-пилот, Computer Use) пишет сюда** баг-репорты по тестированию фаз; **game-lead читает** и чинит.
> game-lead ставит задания на тест в `STATUS.md` с меткой «🟢 ГОТОВО К ТЕСТУ: Фаза N + чеклист».
> Последняя запись — сверху. Формат записи:
> ```
> ## [дата-время] ТЕСТ Фазы N (раунд K) — ветка/коммит
> ИТОГ: PASS / FAIL (X из Y пунктов)
> По пунктам: 1.[PASS/FAIL] <пункт> — ожидалось/получилось [скрин]
> БАГИ (по приоритету): - [критич/средний/мелкий] <описание+воспроизведение+скрин>
> ВОПРОСЫ ДИЗАЙНА (Ринату): …
> НЕ СМОГ ПРОВЕРИТЬ (честно): …
> ```
> game-lead отвечает/реагирует в `STATUS.md`. Дизайн-решения и финальная приёмка фазы — за Ринатом.
