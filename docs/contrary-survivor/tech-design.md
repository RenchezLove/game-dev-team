## 1. Текущая иерархия (сверена живьём, Фаза 0)
```
ACharacter → AMasterHumanoidCharacter → APlayerCharacter           [ЕСТЬ]
APlayerController → AContrarySurvivorPlayerController               [ЕСТЬ]
UActorComponent → UInventoryComponent                              [ЕСТЬ, данные]
AActor → AMasterInventoryItem ┬→ AMasterWeapon → ARangedWeapon → APistol   [ЕСТЬ]
                              └→ AArmor → {AHeadArmor, ATorsoArmor, APantsArmor}  [ЕСТЬ, заглушки]
GameMode = BP_ContrarySurviorGameMode на движковом AGameModeBase    [ЕСТЬ, BP]
HUD = движковый дефолт                                              [ЕСТЬ, дефолт]
```
**GAS НЕ используется** — статы голыми float в персонаже. Решено: кастомный `UStatsComponent` (ADR-015).

- **Ближний бой:** `AMeleeWeapon` `[НОВЫЙ]` (`AMasterWeapon`) — нож. Волк — через `AEnemyCharacter`.

## 3. Принципы реализации (из ADR/GDD)
- **Без GAS** — `UStatsComponent` на float (ADR-015).
- **World Partition отложен** — одна малая карта (ADR-016).
- **Наведение через абстракцию точки экрана** (ADR-017), ПК-клик-захват в Фазе 1.
- **Враги — единым склеенным мешем** (1 draw call); модульный меш — только игрок (GDD ч.2).
- Числа баланса — в `UPROPERTY`/конфиге, крутятся без перекомпиляции логики.
- Каркасы (квесты, модификаторы статов) — расширяемые, но в MVP реализуется минимум.

## 4. Тех-долги к учёту при касании соответствующих систем (из GDD Прил. C)
Перезарядка без таймера; трасса не из MuzzleSocket; броня не влияет на урон; смерть = только лог; орфан-BlendSpace (нет сглаживания анимации по скорости); дубль `BP_PlayerCharacter` на карте; рассинхрон InputAction (нет `IA_Interact`/`IA_Inventory`); именование/`GetMesh()`-затенение/пустой Tick. Не чинить «заодно» вне фазы — заводить как задачу при работе над системой.
