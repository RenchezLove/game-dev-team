# Contrary Survivor — Роудмап по фазам (на утверждение Рината)

> **Версия:** 1.0-draft · **Дата:** 2026-06-12 · **Статус:** НА УТВЕРЖДЕНИИ у Рината (утверждается ОДИН раз, дальше — рабочий ориентир).
> Принципы: **вертикальный срез сначала**, каждая фаза = **играбельный инкремент с пруфом** (реально запускается), диздок=правда. Источник правды по геймплею — `GDD.md`, по классам — `tech-design.md`.
> **Прицел MVP:** играбельный survival-shooter loop к Фазе 5. Каждая фаза проходит **qa-гейт** (сборка+проверка, артефакт) перед merge в master.

## Фаза 1 — Первый вертикальный срез: бой с бандитом 🎯 СЛЕДУЮЩАЯ
**Цель:** на малой карте игрок встречает **одного бандита** с примитивным AI, может его убить, бандит может атаковать игрока; у врага хелсбар; корректная смерть обеих сторон.
**Содержание:**
- `UStatsComponent` (Health + смерть; делегаты).
- `AEnemyCharacter` (бандит — **модульный**, собран из Head/Torso/Legs; ADR-018) + `AEnemyAIController` (заметил→подошёл→атаковал). **Merge модулей в Фазе 1 НЕ делаем** — один враг, живые компоненты; склейка — ~Фаза 3+. Если модули бандита не готовы — временный плейсхолдер-меш (архитектуру не меняет).
- Хелсбар врага (первый UMG-виджет).
- Наведение/клик-захват цели на ПК (абстракция ADR-017).
- Корректная смерть: враг умирает (не просто лог), игрок при 0 HP — заглушка смерти (полноценный респаун — Фаза 2).
- Малая карта: кусок окрестностей деревни (1-2 дома-куба, 1-2 дерева, 1-2 камня) + площадка боя.
**Делегирование:** `cpp-dev` (StatsComponent, EnemyCharacter, AIController, смерть), `unreal-operator` (карта, размещение, BP-связки, хелсбар-виджет), `modeler-3d` (склеенный меш бандита по арту/спеку — или временный плейсхолдер, если арт не готов), `qa` (сборка+прогон).
**Definition of Done:**
- [ ] Проект **компилируется** (лог сборки, qa).
- [ ] При Play: бандит спавнится, замечает игрока, подходит, атакует; игрок стреляет из пистолета, бандит теряет HP (хелсбар реагирует) и **корректно умирает**.
- [ ] Игрок при 0 HP корректно обрабатывается (заглушка смерти, без краша).
- [ ] **Пруф:** видео/скриншоты цикла боя + лог сборки + зелёный qa-вердикт с артефактом.

## Фаза 2 — Выживание, смерть/респаун, сейв, базовый HUD
**Цель:** работает петля выживания и цикл смерть→респаун.
**Содержание:**
- `UStatsComponent`: Hunger/Thirst/Money + деградация + урон при истощении (числа GDD ч.7.3).
- Точки сейва/респауна (`ACampfire`/safe-zone): пара костров + дом. Автосейв + ручной.
- `UContrarySaveGame`: сохранение/загрузка статов+позиции+инвентаря.
- Смерть → респаун на последней точке, потеря % неэкипированного рюкзака.
- Базовый HUD (UMG): HP, голод/жажда при критич., деньги.
**Делегирование:** `cpp-dev` (статы/выживание/сейв/смерть), `unreal-operator` (точки сейва на карте, HUD-виджет, BP), `qa`.
**DoD:**
- [ ] Голод/жажда падают, при критич. — падает HP (наблюдаемо на HUD).
- [ ] Сейв в зоне/у костра → выход → загрузка восстанавливает состояние.
- [ ] Смерть → респаун на точке, часть рюкзака потеряна, экипировка цела.
- [ ] **Пруф:** запись цикла + лог сборки + qa-вердикт.

## Фаза 3 — Бой расширенный: нож, броня, второй враг (волк)
**Цель:** полный боевой ростер MVP и влияние брони.
**Содержание:**
- `AMeleeWeapon` (нож): ближний бой (sweep/overlap, урон, кулдаун).
- Броня влияет на урон (`ArmorProtection` в расчёте) — тех-долг закрыт.
- Второй враг **волк** (через `AEnemyCharacter`, свой меш/поведение — стайность опционально).
- Тех-долги боя по возможности (перезарядка-таймер, трасса из MuzzleSocket).
**Делегирование:** `cpp-dev` (нож, броня→урон, волк-логика), `modeler-3d` (меш волка, меш ножа), `unreal-operator` (размещение, анимации удара/волка), `qa`.
**DoD:**
- [ ] Нож наносит урон в ближнем бою; броня снижает получаемый урон (числа наблюдаемы).
- [ ] Волк спавнится, преследует, атакует, умирает.
- [ ] **Пруф:** запись + лог + qa.

## Фаза 4 — Инвентарь UI, экипировка брони (модульный меш), торговец/экономика
**Цель:** игрок управляет снаряжением и торгует — прогрессия через лут/покупки.
**Содержание:**
- UMG-экран инвентаря (paper-doll слоты брони + рюкзак, drag, деньги).
- Экипировка модульной брони игрока через **Leader/Master Pose Component** (синхронная анимация частей).
- NPC-торговец + экономика (цены GDD ч.7.6), покупка/продажа.
- Подбор лута с врагов/мира.
**Делегирование:** `cpp-dev` (экипировка, экономика, лут), `unreal-operator`/`ui-ux-designer`* (UI инвентаря/торговца), `modeler-3d` (варианты брони, если нужно), `qa`. (*роль добавляется на фазе расширения команды).
**DoD:**
- [ ] Открыть инвентарь, надеть/снять броню — меш персонажа меняется, анимация синхронна.
- [ ] Купить/продать у торговца, деньги меняются.
- [ ] Поднять лут с убитого врага.
- [ ] **Пруф:** запись + лог + qa.

## Фаза 5 — Квесты + диалоги + староста → ИГРАБЕЛЬНЫЙ MVP
**Цель:** замкнутый core loop диздока — исследование → бой → лут → квест → награда → снаряжение.
**Содержание:**
- `UQuestComponent` + структура квеста (типы Kill/Collect/Deliver — в MVP реализован **Kill**).
- Один простейший квест («уничтожить стаю волков») от старосты, награда (деньги/ключи от дома).
- Диалоговое окно (UMG, кнопки-ответы), NPC-староста.
**Делегирование:** `cpp-dev` (квест-каркас, диалоги-логика), `unreal-operator`/`ui-ux-designer` (диалог-UI, NPC), `qa`.
**DoD:**
- [ ] Поговорить со старостой → получить квест → убить волков → счётчик прогресса → сдать → получить награду.
- [ ] **Пруф:** запись полного прохождения квеста + лог + qa.
- [ ] **🏁 MVP-веха:** играбельный сквозной цикл (движение/бой/выживание/инвентарь/торговля/квест/сейв) на ПК.

## Параллельные/сквозные треки (не отдельные фазы)
- **Арт (геометрия/визуал):** Ринат генерит key-art (Прил. E GDD) → вставка в GDD; `modeler-3d` делает меши по артам/спекам по мере фаз. **До конца проекта — препродакшн/плейсхолдеры** (лишь бы читался силуэт и фракция). **Финальные качественные (game-ready) модели — выделенным связным арт-пассом в КОНЦЕ всего проекта** (после лока механик/контента, ориентир — рядом с Фазой 6), чтобы не переделывать поверх меняющихся механик и держать единый стиль (ADR-020). Сюда входит и недоделанный визуал бандита (цвет/границы зон, размер кепки) — отдельного слота не заводим.
- **Инструментарий (тулинг-трек, JIT — ADR-020):** Master-классы оружия (`cpp-dev`, база `AMasterWeapon` есть), Blender prop-киты деревьев/камней/мусора (`modeler-3d`), BP-конструкторы (заборы/брошенные авто/процедурная расстановка — `unreal-operator`, далее `tech-artist`+`level-designer`). Принцип **Just-In-Time:** инструмент/кит делаем, когда фаза ВПЕРВЫЕ упирается в этот класс ассетов и паттерн повторяется (>1 раза) — НЕ заранее «про запас» и НЕ отдельным спринтом. Цель — ускорить разработку и упростить ручное редактирование проекта.
- **Контракт ассета (конвенции):** единый масштаб/пивоты/сокеты, скелет `RootAnim`, топология швов модулей, именование, структура `/Content` — фиксируем РАНО отдельным документом (грунтуется на реальном проекте). Делает поздний финал-арт-пасс дешёвой «подстановкой» (ADR-020).
- **Уборка `TestContentAndCode`** (10 орфанов, GDD Прил. B) — выполнить в удобной точке (рекомендация — перед/в начале Фазы 1, чтобы не тащить мусор). Отдельная мелкая задача `unreal-operator` + qa.
- **Тех-долги** (GDD Прил. C) — закрываются при касании соответствующей системы, не «заодно».

## Процесс на каждой фазе
1. Перед стартом фазы — короткая детализация задач + цитаты из GDD/tech-design в постановке специалистам.
2. ≤3 специалиста параллельно, каждый в своей фича-ветке/worktree.
3. Сбор результатов с пруфами → синтез → **qa-гейт** (сборка+тест зелёные, артефакт) → `.claude/QA_OK` → merge `--no-ff` → снять маркер.
4. Отчёт Ринату: что сделано, чем подтверждено, что осталось.
