# Архитектура game-dev-team

## Иерархия
```
game-lead                 (руководитель, главная сессия: claude --agent game-lead)
   ├── cpp-dev            (C++ программист, UE 5.5)
   ├── unreal-operator    (оператор Unreal Engine)
   ├── modeler-3d         (3D-моделлер, Blender → UE)
   ├── concept-artist     (концепт-художник)
   └── qa                 (сборка/тесты, мерж-гейт)
```
Расширение (следующие фазы): game-designer, tech-artist, sound, level-designer, ui-ux-designer.

## Структура репозитория
```
game-dev-team/
├── CLAUDE.md                 # корневая память: правила, стек, делегирование, анти-галл. протокол
├── .claude/
│   ├── agents/               # определения агентов (.md + YAML frontmatter)
│   └── settings.json         # настройки проекта (заполняется на Фазе автономии)
├── context/
│   ├── _shared.md            # общий слой: стек, конвенции, инструменты, потоки
│   └── <agent>/context.md    # персональная дистилляция состояния по каждому агенту
├── docs/
│   ├── architecture.md       # этот файл
│   └── decisions.md          # журнал ADR
└── logs/                     # логи прогонов
```

## Принципы
- **Плоское дерево.** Субагент не может порождать субагентов; цепочки строит `game-lead` из главной сессии.
- **Нативное первым** при выборе инструментов; решения — в `docs/decisions.md`.
- **Анти-галлюцинационный протокол** (см. `CLAUDE.md`) обязателен для всех.
- **Источник истины по игре** — дизайн-документ проекта (грузится по запросу).
