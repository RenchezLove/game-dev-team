# _shared.md — общий слой команды

> Стек, конвенции и потоки работы, общие для всех агентов. Читается агентами при работе над задачей. Технические детали по версиям — всегда сверять с официальной документацией (см. анти-галл. протокол в `CLAUDE.md`).

## Технологический стек
- **Движок:** Unreal Engine 5.5.
- **Код:** C++ (через Visual Studio 2022 + workload «Game Development with C++») + Blueprints.
- **3D:** Blender (моделирование, ретопология, UV, материалы, анимации) → экспорт в UE.
- **Платформы:** ПК (основная) + Android (цель — слабые устройства). iOS отложен.
- **VCS:** git + приватный GitHub-репозиторий проекта. Бэкап крупных ассетов — Google Drive.

## Поток работы
1. Ринат ставит задачу `game-lead`.
2. `game-lead` декомпозирует, цитируя источник истины (диздок / своя постановка), делегирует ≤3 специалистам.
3. Специалист выполняет в своей зоне, возвращает результат + доказательства (логи/файлы/git).
4. `game-lead` синтезирует, при необходимости запускает следующий шаг, отдаёт `qa` на гейт.
5. `game-lead` отчитывается Ринату; нерешаемые/дизайнерские вопросы — эскалирует Ринату.

## Зоны ответственности (кратко)
- **game-lead:** декомпозиция, делегирование, синтез, интеграция/мерж, отчёт и эскалация.
- **cpp-dev:** игровой код C++ под UE 5.5, сборка кода, доказательство компиляции.
- **unreal-operator:** работа в редакторе UE — сцены, ассеты, блюпринты, сборка проекта.
- **modeler-3d:** 3D-ассеты в Blender, экспорт в UE-совместимом формате.
- **concept-artist:** мудборды/стайлфреймы/наброски, ТЗ на ассеты для modeler-3d.
- **qa:** прогон сборки и тестов, проверка на реальных артефактах, мерж-гейт.
